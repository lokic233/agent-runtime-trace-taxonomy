## Oracle-gap interpretation (canonical)
**No positive lower bound on real oracle headroom can be established from the current single-run
matrix**, under any of 7 tested noise models (6/7 manufacture a null gap ≥ observed; the one exception
requires an unmeasurable within-task noise correlation). This is **NOT** the same as "the true oracle
gap is exactly zero." The result is a methodological negative about *identifiability*, not a proof of
no-headroom.

## Why the earlier wording was too strong
"No useful parity" / "bias-corrected gap ~0%" / "0/50 tasks show a stable crossing" read as *proof*
that no parity exists. The evidence only supports *non-identifiability from this matrix*: 0/50 tasks
have **repeated evidence** to establish a crossing (1 rep/cell), which is different from 50/50 tasks
**proven** to have no crossing. See PARITY_STUDY_SCOPE_RECONCILIATION.md.

## Next step
Do NOT resume signal learning on the retrospective matrix. The smallest fair next experiment is a
**staged, noise-calibrated, replicated pilot on a genuinely parity-bearing action family (agentic
memory)** — designed (not run) in AGENTIC_MEMORY_PARITY_PILOT_DESIGN.md. A precise scoped negative
plus a fair next-study design is the intended outcome.
